# Архитектура отображения параметров калькулятора

## Зачем разделяем обозначения

В калькуляторах Calc Square одни и те же размеры участвуют сразу в нескольких контекстах. Если показывать пользователю только внутренние CAMduct-ключи, например `A` и `B`, интерфейс становится похожим на техническую карту импорта, а не на понятный калькулятор изделия.

Нужно разделять:

- CAMduct keys: `A`, `B`, `C`, `D` и другие внутренние ключи CAMduct.
- Geometry symbols: `ØD`, `D`, `L`, `t`, `W`, `H` и другие геометрические обозначения.
- User UI labels: `1. Диаметр ØD`, `2. Длина L`.
- Visual roles: `diameter`, `length`, `height`, `thickness`, `connector`.

Главное правило: CAMduct-ключи остаются стабильными внутренними ключами для расчётов, сохранённых позиций, импорта и экспорта. Пользовательский display-layer строится поверх них.

## Четыре слоя

### CAMduct layer

Слой внутренних ключей и соответствий CAMduct.

Примеры:

- `camductKey: A`
- `camductKey: B`
- `camductKey: C1`
- `camductKey: C2`

Этот слой нельзя ломать переименованием ради визуального удобства.

### Geometry layer

Слой инженерной геометрии изделия.

Примеры:

- `ØD` или `D` — диаметр.
- `L` — длина.
- `t` — толщина.
- `W` — ширина.
- `H` — высота.

Geometry layer используется в понятных пользователю подписях, спецификации и эскизах.

### UI layer

Слой пользовательского интерфейса.

Примеры:

- `1. Диаметр ØD`
- `2. Длина L`

UI layer может иметь номер параметра, название, короткий geometry symbol и подсказку. Он не обязан показывать CAMduct key обычному пользователю.

### Visual layer

Слой эскиза и визуальных связей.

Примеры:

- цветная линия диаметра;
- цветная линия длины;
- маркер возле поля ввода;
- легенда параметров;
- подсветка размерной линии при фокусе поля.

Visual layer не должен перегружать сам эскиз длинными текстами.

## Режимы отображения

### USER mode

Обычный пользователь видит только понятные геометрические обозначения и названия параметров.

Правила:

- CAMduct `A` / `B` / `C` / `D` скрыты.
- Поля показываются в пользовательском формате:
  - `1. Диаметр ØD`
  - `2. Длина L`
- Строка спецификации может показывать geometry format:
  - `ØD 250 × L 3200`
- CAMduct mapping хранится невидимо.

### ADMIN / CAMduct mode

Администратор или проверяющий может включить отображение CAMduct-обозначений.

Правила:

- Включается отдельной кнопкой или переключателем, например `Показать CAMduct-обозначения`.
- Режим влияет только на отображение.
- Расчёты, export/import, `dimensions` и `rawDimensions` не меняются.
- При включении рядом с параметрами показывается связь:
  - `CAMduct A → ØD`
  - `CAMduct B → L`
- На эскизе CAMduct-обозначения показываются не поверх изделия, а в легенде или компактном overlay-блоке.

### DEBUG / mapping mode

Режим зарезервирован для будущей полной проверки связей.

Он может показывать цепочку:

`UI index → CAMduct key → geometry symbol → formula symbol`

Пример:

`1 → A → ØD/D → D`

## Mapping R-001 / Воздуховод круглый

### Параметр 1

- internal key: `A`
- camductKey: `A`
- geometrySymbol: `ØD` / `D`
- uiIndex: `1`
- uiLabel: `Диаметр`
- formulaSymbol: `D`
- visualRole: `diameter`
- colorToken: `dimension-diameter`

### Параметр 2

- internal key: `B`
- camductKey: `B`
- geometrySymbol: `L`
- uiIndex: `2`
- uiLabel: `Длина`
- formulaSymbol: `L`
- visualRole: `length`
- colorToken: `dimension-length`

## Правила сохранения совместимости

- Не переименовывать внутренние ключи `A` / `B` в коде расчёта.
- Не менять `rawDimensions` / `dimensions` для CAMduct-совместимости.
- Не ломать сохранённые позиции проекта.
- Display-layer делать поверх существующих ключей.
- Расчёт площади R-001 продолжает получать значения через `values.A` и `values.B`.
- Preview может получать совместимость `D = A`, `L = B`, но это не должно менять внутренние ключи.

## Правила обычного пользовательского UI

- Не показывать CAMduct `A` / `B` / `C` / `D` обычному пользователю.
- Показывать понятные номера и геометрические обозначения:
  - `1. Диаметр ØD`
  - `2. Длина L`
- В спецификации разрешён geometry format:
  - `ØD 250 × L 3200`
- CAMduct mapping хранится невидимо.
- Если нужен доступ к CAMduct-связям, он включается только через ADMIN / CAMduct mode.

## Правила ADMIN / CAMduct mode

- Включается отдельной кнопкой или переключателем: `Показать CAMduct-обозначения`.
- Режим влияет только на отображение.
- Расчёты, export/import, `dimensions` и `rawDimensions` не меняются.
- Рядом с параметрами можно показывать:
  - `CAMduct A`
  - `CAMduct B`
- В легенде можно показывать:
  - `CAMduct A → ØD / Диаметр`
  - `CAMduct B → L / Длина`
- На эскизе CAMduct-обозначения не размещать поверх изделия.

## Правила для эскизов

- На аксонометрии минимум текста.
- Подписи не должны накладываться на изделие.
- Основные обозначения показывать в легенде.
- Использовать дополнительные виды к аксонометрии:
  - вид сбоку;
  - вид спереди / торец;
  - вид сверху.
- В USER mode показывать `ØD` / `L`.
- В ADMIN mode дополнительно показывать `CAMduct A` / `CAMduct B` в легенде.
- Размерные линии должны быть короткими, цветными и легко сопоставимыми с полями ввода.

## Цветовая система

Цвета должны назначаться не буквам CAMduct, а визуальным ролям.

- `dimension-diameter` — diameter / section.
- `dimension-length` — length / axis.
- `dimension-height` — height / depth.
- `dimension-thickness` — thickness.
- `dimension-connector` — connectors.

Для R-001:

- `A` → `ØD` → `dimension-diameter`.
- `B` → `L` → `dimension-length`.

## Минимальное правило для реализации

Любое изменение отображения параметров должно отвечать на три вопроса:

1. Какой внутренний ключ остаётся в расчёте и сохранённых данных?
2. Какое geometry symbol видит пользователь?
3. Как ADMIN / CAMduct mode показывает связь между ними?

Если изменение требует переименовать внутренний ключ ради UI, такое изменение считается небезопасным.
