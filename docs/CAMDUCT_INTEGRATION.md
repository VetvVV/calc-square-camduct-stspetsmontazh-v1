# CAMduct integration base

Цель: связать строки Calc Square со структурой CAMduct так, чтобы проект можно было сохранить один раз и дальше не набирать заново в CAMduct.

## Что уже заложено

- `assets/camduct/camduct-map.js` хранит словарь соответствий: наше изделие, CAMduct family, картинка, стандарт, материал и параметры.
- `home.html` сохраняет в `calc-square-spec.json` дополнительный блок `camduct` у каждой строки.
- Кнопка `CAMduct JSON` выгружает отдельный файл `calc-square-camduct.json` для анализа и будущего конвертера.
- Кнопка `Импорт MAJ` читает `.MAJ` в браузере, ищет известные CAMduct-токены и добавляет найденные изделия в спецификацию как черновик.
- `scripts/analyze-camduct-maj.ps1` извлекает читаемые строки из `.MAJ` и считает частоты.

## Почему не пишем MAJ сразу

`.MAJ` является бинарным job-файлом CAMduct. При открытии CAMduct может создавать временный `.MLK`; его нужно считать служебным/lock-файлом и не использовать как основной формат обмена.

Писать `.MAJ` безопасно можно только после сравнения нескольких реальных проектов: один и тот же набор изделий, разные размеры, материалы, толщины и количества. Поэтому текущий этап — собрать словарь и понять позиции полей.

## Обратный импорт из MAJ

Импорт из `.MAJ` сейчас работает как черновое восстановление спецификации:

- файл выбирается кнопкой `Импорт MAJ`;
- приложение извлекает читаемые UTF-16 строки прямо в браузере;
- известные картинки/семейства из `camduct-map.js` превращаются в строки спецификации;
- размеры и количество нужно проверять вручную, потому что точная бинарная структура `.MAJ` ещё не размечена.

Каждая импортированная строка хранит `rawDimensions.camductContext` — кусок исходного CAMduct-контекста. По этим контекстам можно уточнять правила распознавания.

## Подтверждённые эталоны

- `test_round_elbow_90_d125.MAJ` и `test_round_elbow_90_d250.MAJ`: отличаются только строкой диаметра `125`/`250` и именем файла. Для `RND-SegmentBend.png` можно брать `D` из числовой строки рядом с маркером, а угол из текста `отвод 90`.
- `test_round_elbow_45_d125.MAJ`: отличается от `test_round_elbow_90_d125.MAJ` только строкой `отвод 45--`. Значит угол круглого отвода можно брать из названия изделия.
- `test_round_elbow_90_d125_qty2.MAJ` и `test_round_elbow_90_d125_qty3.MAJ`: количество не видно в читаемых строках, но подтверждено бинарно. Для `RND-SegmentBend.png` количество читается как `Int32LE` по смещению `marker + 974`: `1`, `2`, `3`.
- Для этих эталонов также читаются `GALVANISED`, `GALV`, `точка0/3`, `8/8`, `DW144-LV`, `HVAC`.

## Обозначения CAMduct: круглый отвод

Для `RND-SegmentBend.png` на вкладке размеров CAMduct:

- `A` — диаметр;
- `B` — внутренний радиус;
- `C` — угол;
- `D` — нижнее удлинение;
- `E` — верхнее удлинение.

На вкладке соединителей:

- `C1` — соединитель `8/8` для стандартного отвода;
- `S1` — соединитель/шов `точка0/3`;
- `D1`, `D2` — дополнительные соединители, в эталонах `Нет`.

## Обозначения CAMduct: круглый воздуховод

Для `RND-StraightSeamedPipe.png` на вкладке размеров CAMduct:

- `A` — диаметр;
- `B` — длина;
- `C` — левое удлинение;
- `D` — правое удлинение.

На вкладке опций:

- `SeamPosition` — расположение шва, в эталоне `90,0`;
- `MaterialType` — тип диаметра, в эталоне `Номинальный`.

На вкладке соединителей:

- `C1`, `C2` — торцевые соединители, в эталоне `Нет`;
- `S1` — продольный шов, в эталоне `12,5/12,5`;
- `D1`, `D2` — дополнительные соединители, в эталоне `Нет`.

В `test_round_duct_d125_l1000.MAJ` значения `125` и `1000` не читаются текстом рядом с маркером, они лежат бинарно.

- `test_round_duct_d250_l1000.MAJ`: подтверждает изменение диаметра.
- `test_round_duct_d125_l1500.MAJ`: подтверждает изменение длины.
- Для `RND-StraightSeamedPipe.png` параметры читаются как `DoubleLE` относительно маркера картинки: `A = marker - 952`, `B = marker - 1004`.

## Стандартные изделия из CAMduct

Если при расшифровке изделие помечено как стандартное, это означает: параметры уже получены из CAMduct и не должны вводиться вручную.

- `dimensions` и `camductOriginalParams` хранят расшифрованные параметры для калькулятора.
- Калькулятор должен открываться с этими параметрами как стартовыми.
- Если пользователь ничего не менял, `camductDirty=false`, и обратный CAMduct-экспорт использует исходные стандартные параметры.
- Если пользователь изменил размер, угол, материал или соединение, строка должна стать `camductDirty=true`, а в экспорт пойдут обновлённые параметры.

## Как анализировать готовые проекты

Один файл:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\analyze-camduct-maj.ps1 -Path "C:\Users\vyach\Desktop\89139.MAJ"
```

Папка с проектами:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\analyze-camduct-maj.ps1 -Path "D:\CAMductSamples"
```

Результат по умолчанию пишется в `camduct-analysis/`:

- `index.json` — список обработанных файлов;
- `*.summary.json` — найденные CAMduct-токены и частотный список;
- `*.strings.csv` — таблица строк и количества повторений.

## Какие примеры полезнее всего

Лучше всего давать маленькие CAMduct-проекты, где понятно, что внутри:

- один круглый отвод 90 на D125;
- такой же отвод на D250;
- круглый воздуховод с известной длиной;
- прямоугольный канал A/B/L;
- переход с известными диаметрами;
- проект с теми же изделиями, но другим материалом или толщиной.

Так проще увидеть, какие байты и строки меняются при изменении одного параметра.
